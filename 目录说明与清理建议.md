# 目录说明与清理建议

这份说明专门用来回答两个问题：

1. 哪些目录是项目本体，不能随便删？
2. 哪些目录只是测试、缓存、烟测产物，可以清理？

## 建议保留

| 目录 | 作用 | 建议 |
| --- | --- | --- |
| `codex-skill` | 给 Codex 安装 skill 的源码目录 | 保留 |
| `data` | `backend` 模式的作业数据、状态、事件日志 | 通常保留 |
| `review_outputs` | 真实审稿输出目录，里面是你的结果文档 | 通常保留 |
| `src` | 项目主代码 | 保留 |
| `tests` | 自动化测试代码 | 保留 |

## 可以删除

| 目录 | 作用 | 建议 |
| --- | --- | --- |

| `pytest-tmp-clean` | 我之前跑回归测试时的临时目录 | 可删 |
| `skill-smoke` | 我做 skill 烟测时生成的临时目录 | 可删 |
| `skill-smoke/_normalize_cache` | `skill-smoke` 里的归一化缓存 | 可删 |
| `skill-smoke/sample-paper` | `skill-smoke` 里的示例产物 | 可删 |
| `tmp-codex-check` | 调试 Codex 运行链路时的临时目录 | 可删 |
| `tmp-codex-naming-check` | 调试命名规则时的临时目录 | 可删 |
| `tmp-debug2` | 临时调试目录 | 可删 |
| `tmp-debug3` | 临时调试目录 | 可删 |
| `tmp-debug4` | 临时调试目录 | 可删 |
| `tmp-naming-check` | 调试结果命名时的临时目录 | 可删 |
| `tmp-norm-check` | 调试归一化流程时的临时目录 | 可删 |
| `tmp-skill-prepare-check` | 调试 skill 预处理时的临时目录 | 可删 |

## 为什么会有这些目录

- `.pytest_cache`、`.tmp`、`pytest-tmp-clean`：跑测试时生成
- `skill-smoke`、`tmp-*`：我在实现和验证功能时做烟测/调试时生成
- `review_outputs`：正常审稿输出目录
- `data`：`backend` 模式的作业目录
- `.basetemp`、`basetemp_plain`、`pytest-cache-files-*`：之前 Windows 权限异常留下来的坏目录

## 怎么清理

### 普通清理

适合清掉绝大多数临时目录，不会碰你的正式结果：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\清理临时文件-普通模式.ps1
```

### 管理员模式清理

适合连历史坏目录一起强制删掉：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\清理临时文件-管理员模式.ps1
```

或者直接双击：

`一键清理-管理员模式.bat`

## 当前推荐

如果你只是想把项目目录先清爽一下：

1. 先运行 `清理临时文件-普通模式.ps1`
2. 如果还剩 `.basetemp`、`basetemp_plain`、`pytest-cache-files-*` 这些目录，再运行管理员模式脚本

## 不建议随便删除的目录

除非你明确知道不要这些结果了，否则不要随便删：

- `review_outputs`
- `data`
- `codex-skill`
- `src`
- `tests`
