fusion-reviewer

上传论文后，系统会先做一次共享证据预处理，再进入 backend 模式或 Codex skill 模式的委员会审稿流程。PDF 会优先走 MinerU，高保真失败时再回退到本地结构化解析；Word 默认先转 PDF 再审。

命令行

使用 fusion-review submit --paper paper.pdf 跑 backend；加上 --mode codex 则只准备 Codex skill 需要的运行目录。

共享证据

PDF、DOCX、DOC 都先走统一预处理层。优先 MinerU，其次本地结构化 fallback，最后才是纯文本兜底。

产物目录

每次运行都会写入类似 review_outputs/20260327-214500__论文名__短任务号 这样的清晰目录，里面会有 evidence、8 份 reviewer 结果、concerns 表、meta review、final report 等文件。
